# Language Rules

- Luôn trả lời người dùng bằng tiếng Việt.
- UI text, explanation, requirements, design, tasks phải viết bằng tiếng Việt.
- Nếu cần dùng EARS notation, được giữ từ khóa WHEN / THE SYSTEM SHALL, nhưng phần mô tả phải là tiếng Việt.
- Không tự chuyển sang tiếng Anh trừ khi tên code, tên class, tên function, package, command hoặc thuật ngữ kỹ thuật bắt buộc.
- Khi tạo Spec, ưu tiên ngắn gọn, rõ ràng, không hỏi nhiều câu nếu có thể tự suy luận.
